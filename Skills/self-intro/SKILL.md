# Self Intro（中文面试自我介绍）

## 目标

在**信息不完整**的情况下，也能稳定产出一段**约 60 秒**、适合口述的中文面试自我介绍，并确保：

- 逻辑清晰（我是谁 → 我做过什么 → 我擅长什么 → 我想要什么）
- 信息密度高（项目/成果尽量量化）
- 贴合岗位（技术栈与职责对齐）
- 语气职业化但自然，不“背稿味”

## 使用时机（触发词）

当用户提出以下诉求时使用本技能：

- “帮我写自我介绍”“面试自我介绍”“介绍一下你自己”“30/60 秒电梯稿”“面试开场白”
- “把我这段经历润色成自我介绍”“根据 JD 写自我介绍”

## 工作流程

1. **快速采集信息（缺什么补什么）**
   - 如果用户没有提供某字段，用**最少问题**补齐；优先问“岗位方向/JD”“1-2 个代表项目”“量化结果”。
2. **岗位对齐**
   - 从用户信息中挑出与岗位最相关的 2-3 个关键词（例如：推荐/检索、LLM 应用、后端高并发、数据平台等），贯穿全文。
3. **生成 60 秒标准版（默认输出）**
   - 结构固定：开场一句 + 背景/年限 + 技术栈/领域 + 1-2 个项目亮点（量化）+ 优势/工作方式 + 求职诉求/匹配度。
4. **自检与微调**
   - 去掉空泛形容词（“熟练掌握/精通/擅长”），用事实替代。
   - 每段 1-2 句，避免超长句；口语化但不随意。

## 信息采集清单（按优先级）

优先拿到这些信息再写（拿不到也要先写出草稿，并明确假设）：

- **姓名/称呼**：例如“我叫张三/你可以叫我 Alex”
- **求职方向/目标岗位**：例如“后端工程师（Go）/算法工程师（推荐）/LLM 应用工程师”
- **经验年限/背景一句话**：例如“3 年后端”“硕士 + 2 年算法”
- **技术栈/擅长领域（3-6 个关键词）**
- **代表项目亮点（1-2 个）**
  - 项目是什么、你负责什么、用了什么技术、结果/指标是什么（至少给一个可量化点）
- **优势/特点（2 条以内）**：例如“端到端推进”“工程化与数据结合”
- **价值观/工作方式（1 句）**：例如“结果导向 + 关注可观测性与稳定性”
- **当前诉求（1 句）**：例如“希望做更偏业务闭环的检索/推荐”“想去做 ToB 工程落地”

## 默认输出模板（60 秒）

按下面模板生成成稿（避免小标题，直接成段口述）：

```text
你好，我叫【姓名/称呼】，目前主要方向是【求职方向/目标岗位】。我有【年限/背景一句话】的经验，主要在【领域/行业或场景】做过【一句话概括工作内容】。

技术上我比较熟悉【技术栈关键词 1-3 个】，也做过【补充关键词 1-2 个】相关的工程落地。比较有代表性的经历有两点：第一，【项目A：你负责的核心工作 + 技术点】；最终带来【量化结果/业务指标/效率提升】。第二，【项目B：你负责的核心工作 + 技术点】；结果是【量化结果】。

我的优势是【优势 1】和【优势 2】；平时做事比较【价值观/工作方式一句话】。我这次希望应聘【岗位】，主要看重【诉求/匹配点】，也相信我在【关键词】上的经验能比较快在团队里产生价值。谢谢。
```

## 语言与风格规则

- **句式**：偏口语化，避免“首先/其次/最后”堆砌；每句话尽量不超过 25-30 个字。
- **量化**：优先写“指标/规模/成本/时延/转化/稳定性/人效”。
- **克制**：除非用户明确要求，不写“精通/专家/顶尖”等夸张词。
- **对齐**：当用户提供 JD 时，把 JD 的 3-5 个关键词融合进“技术栈/项目亮点/诉求”。

## 可选变体（按需输出）

如果用户明确说“更短/更长/更偏技术”，在不改变事实的前提下改写：

- **30 秒电梯版**：只保留“方向 + 年限 + 1 个项目亮点 + 诉求”
- **2 分钟扩展版**：每个项目多补 1 句“难点与解决方案/取舍”
- **偏技术版**：突出架构、性能、可靠性、工程化方法；少写行业背景

## 示例（占位）

当用户给出信息后，生成与其事实一致的成稿；如信息不足，先给草稿并列出 3 个最关键的补充问题。
